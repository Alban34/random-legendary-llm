# Epic 49 — Clear Selection Regression Fix & E2E Guard: Task List

## Story 49.1 — Identify and fix the Clear Selection regression in the setup/generator component

- [ ] Read `src/app/state-store.mjs` and confirm the root cause: `clearActiveSetIds(state)` sets `collection.activeSetIds = null`; in `NewGameTab.svelte` the `isChecked` expression is `appState.collection.activeSetIds === null || appState.collection.activeSetIds.includes(setId)`, so `null` renders every checkbox as **checked** (all-owned); because both the `data-action="active-filter-select-all"` and `data-action="active-filter-clear-all"` buttons call the same `gameActions.clearActiveSetIds` handler, clicking “Clear selection” never actually unchecks any checkbox.
- [ ] In `src/app/state-store.mjs`, add a new exported function `deactivateAllSets(state)` that returns a deep-cloned state with `collection.activeSetIds` set to `[]` (empty array); this makes every `isChecked` expression evaluate to `false`, unchecking all checkboxes in the UI.
- [ ] In `src/components/App.svelte`, import the new `deactivateAllSets` function from `../app/state-store.mjs` (aliased as `deactivateAllSetsStore` to avoid name collision), then add a `deactivateAllSets()` entry to the `actions` object that calls `applyStateUpdate((s) => deactivateAllSetsStore(s), locale.t('actions.clearedActiveFilter'))`.
- [ ] In `src/components/App.svelte`, expose the new action in the `gameActions` object (lines ~1134–1148) by adding `deactivateAllSets: actions.deactivateAllSets` alongside the existing `clearActiveSetIds` entry.
- [ ] In `src/components/NewGameTab.svelte`, change the `onclick` of the `data-action="active-filter-clear-all"` button (currently at line ~159) from `{gameActions.clearActiveSetIds}` to `{gameActions.deactivateAllSets}`; leave the `data-action="active-filter-select-all"` button unchanged — it correctly calls `gameActions.clearActiveSetIds` (sets `null` = all selected).
- [ ] In `test/playwright/epic46-active-filter.spec.mjs`, update the existing test `'"Clear selection" also restores the all-owned state'` to reflect the now-correct behavior: after clicking `[data-action="active-filter-clear-all"]`, assert that every `[data-active-filter-checkbox]` is **unchecked**; update the test description to `'"Clear selection" unchecks all expansion checkboxes'`.
- [ ] Run `npm run lint` and fix any ESLint errors introduced in `state-store.mjs`, `App.svelte`, and `NewGameTab.svelte` before proceeding.
- [ ] **Test:** In a browser with multiple expansions owned, navigate to the New Game tab; confirm the Active Filter panel is visible with all expansion checkboxes checked; click “Clear selection” (`data-action="active-filter-clear-all"`); assert every `[data-active-filter-checkbox]` is now visually unchecked; confirm the summary updates to reflect zero active expansions; click “Use all expansions” (`data-action="active-filter-select-all"`) and confirm all checkboxes return to checked and the summary reads “All N expansions”.
- [ ] **QC (Automated):** Run `npm run lint`; confirm zero errors in `src/app/state-store.mjs`, `src/components/App.svelte`, and `src/components/NewGameTab.svelte`; run `npx playwright test test/playwright/epic46-active-filter.spec.mjs` and confirm all tests pass with the updated assertion.
