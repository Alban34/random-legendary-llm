## What's in v1.0.1

This is a polish and bugfix release addressing issues reported after the v1.0.0 launch.

---

### Bug fixes

**Score input focus** — After typing the first digit of a score, focus no longer jumps away from the input field. You can now type a full number without having to click back in.

**French translations restored** — All French UI text was missing accents and apostrophes (e.g. `'`, `é`, `è`). Every affected string has been corrected. Other languages were audited at the same time.

**"Non possédée actuellement." corrected** — The French placeholder text for unowned sets was displaying garbled text. It now reads correctly.

**Missing translations filled in** — Several user-facing strings that were previously displayed in English regardless of the selected language have been translated across all six supported locales.

**Local development server fixed** — Running the app locally was generating four 404 errors in the browser console and causing the page to not display. This is resolved without affecting the GitHub Pages deployment.

---

### UI and header improvements

**Version number pulled from `package.json`** — The version shown in the header is now read directly from the project's `package.json`, so it will always stay in sync.

**Cookie/storage disclosure added** — A notice about local storage use has been added to satisfy transparency expectations.

**GitHub repository link added** — A GitHub icon now appears at the far right of the header, linking to the source repository.

**Compact language and theme controls** — The language selector and dark/light theme toggle previously took up too much header space. They have been replaced with flag and icon controls that are more compact.

**Mobile header layout** — On mobile viewports, the new compact icons sit on the same line as the app title, saving vertical space.
