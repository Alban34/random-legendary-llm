## What is this?

Legendary: Marvel Randomizer is a free, browser-based companion app for players of *Legendary: A Marvel Deck-Building Game*. Open the URL. Get a legal, fresh, collection-aware game setup in seconds. No install. No account. No tracking. Ever.

**[Try it now →](https://alban34.github.io/random-legendary-llm)**

---

## What's in v2.0.0

v2.0.0 is a significant release on two fronts. The first front is infrastructure: all runtime source code has been migrated from JavaScript (`.mjs`) to TypeScript (`.ts`), adding compile-time safety across every module. The second front is features: Epic Mastermind mode, Forced Hero Team, Preferred Expansion Priority, improved toast notifications, solo-mode rule corrections, UI polish, and two data and rule bug fixes are all included. There are no breaking changes to your saved data — existing backups and in-browser state are fully forward-compatible.

---

### Full TypeScript source migration (Epics 61–68)

All runtime modules under `src/app/` have been migrated from `.mjs` to `.ts`. A single authoritative type file (`src/app/types.ts`) declares all domain types: game data shapes, runtime index types, application state types, setup output types, and import utility return types. All modules import shared types from this file rather than re-declaring them locally.

Migrated modules include `state-store.ts`, `setup-generator.ts`, `setup-rules.ts`, `solo-rules.ts`, `backup-utils.ts`, `collection-utils.ts`, `localization-utils.ts`, `bgg-import-utils.ts`, `myludo-import-utils.ts`, `theme-utils.ts`, `history-utils.ts`, `object-utils.ts`, `focus-utils.ts`, `game-data-pipeline.ts`, `app-renderer.ts`, `browser-entry.ts`, and all six per-locale catalog files (`en.ts`, `fr.ts`, `de.ts`, `ja.ts`, `ko.ts`, `es.ts`).

---

### `tsx` test runner

The unit test runner now uses `node --import tsx/esm --test` (via the `tsx` package) instead of `node --experimental-strip-types --test`. This allows test files to import directly from `.ts` source modules without any pre-compilation step. All `test:*` npm scripts and the `test:coverage` script have been updated accordingly. The `code-analysis` CI job continues to produce a valid LCOV report for SonarCloud.

---

### `svelte-check` type gate in lint

`svelte-check --tsconfig ./tsconfig.json` is included in the `npm run lint` script. This ensures that every CI lint pass also validates TypeScript type correctness across all Svelte components and runtime modules. Lint failures are blocking.

---

### No user-facing data schema changes

The persisted data schema (`legendary_state_v1`) is unchanged. No migration is required. All backups created with any previous version of the app remain fully compatible with v2.0.0.

## New features

### Epic Mastermind difficulty mode

Players who own the **X-Men** expansion can now opt into **Epic Mastermind** mode directly from the New Game tab. When the toggle is enabled, the generator restricts the mastermind pool to cards with the Epic Mastermind designation — delivering a harder, campaign-grade challenge. The toggle only appears in the DOM when you own at least one supporting expansion; it is completely absent otherwise. The Epic Mastermind flag is stored on every history record so your win/loss tracking stays accurate. The toggle state persists across page reloads. More expansions will be added to the supported set as their Epic Mastermind cards are catalogued.

---

### Forced Hero Team

The forced-picks panel now includes a **Forced Team** selector. Choose a team affiliation (e.g. X-Men, Avengers, S.H.I.E.L.D.) and the generator will draw heroes from that team first, within the normal freshness-priority rules. If the team pool is smaller than the required hero count, remaining slots fill from the general pool automatically. Individually forced hero IDs still take absolute priority. The active forced team is shown as a pill in the generated setup result view, and the selection is included in backup exports so it round-trips correctly. The selector is hidden when no heroes in the active collection carry a team affiliation.

---

### Preferred Expansion Priority

In the forced-picks panel you can now pin one owned expansion as **Preferred**. When the generator selects cards for unclaimed slots, it draws from the preferred expansion first within each play-count freshness tier — without overriding individually forced picks or violating any legality constraint. If the preferred expansion has no eligible cards of the required type in the current tier, the generator falls back silently to the full pool. The selector is hidden when you own fewer than two expansions. The preference is included in backup exports.

---

### Improved toast notifications (svelte-sonner)

The custom in-app notification system has been replaced with [svelte-sonner](https://svelte-sonner.vercel.app/). Toasts now stack with a spring animation, auto-dismiss after four seconds, and use colour-coded variants (success, info, warning, error) with richer visual contrast. Persistent error and warning toasts (those that require acknowledgement) remain displayed until dismissed. Manual timer management and the `ToastStack` component have been removed; the library handles all lifecycle internally.

---

### Active Expansions section collapsed by default

The **Active Expansions** section on the New Game tab is now collapsed when you first open the page. The section header shows a toggle button with a correct `aria-expanded` attribute; one tap expands or collapses the panel. The expanded/collapsed state is preserved across tab navigation within the same session (the component is never unmounted on tab switch, so the local state persists naturally).

## Bug fixes and corrections

### Solo mode: "Always Leads" villain group no longer forced

In all solo play modes (Standard Solo, Advanced Solo, Two-Handed Solo, Standard Solo v2), the mastermind's "Always Leads" villain or henchman group is no longer forced into the setup. Previously, the generator treated the lead as a mandatory slot assignment regardless of play mode, leaving one fewer free slot for random selection. The lead entity is now `null` in solo mode; the result view shows "No mandatory lead" for the mastermind, and all villain and henchman group slots are drawn freely from your collection. Non-solo play modes are unaffected — the "Always Leads" rule still applies at all multiplayer player counts.

### Ant-Man and the Wasp correctly classified as a small expansion

The **Ant-Man and the Wasp** set was incorrectly classified as a large expansion in the game data. It is now correctly classified as a small expansion. This affects how the set appears in the Browse tab type filter and the Collection tab grouping, but does not change any card counts or rule constraints.
