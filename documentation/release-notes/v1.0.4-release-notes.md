### Safari on iOS/iPadOS: home screen icon fixed

When adding the app to the home screen via **Share → Add to Home Screen** in Safari on iPhone or iPad, the icon was displaying as a screenshot of the page rather than the proper app icon. The root cause was an absolute path (`/icons/icon-192.png`) in the `apple-touch-icon` link tag, which resolved to the domain root and missed the `/random-legendary-llm/` sub-path the app is deployed under. The path has been corrected to the base-URL-aware form (`%BASE_URL%icons/icon-192.png`), consistent with the manifest link. Safari on iOS now shows the correct icon on the home screen.

> **Note on iOS/iPadOS install UX:** Safari does not display an automatic install banner or prompt. To add the app to your home screen, tap the **Share** button (box with upward arrow) in the Safari toolbar and select **Add to Home Screen**. Chrome and other third-party browsers on iOS are subject to Apple's WebKit restriction and do not support PWA installation on that platform regardless of app configuration.
