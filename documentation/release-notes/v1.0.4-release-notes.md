Legendary: Marvel Randomizer is a free, browser-based companion app for players of *Legendary: A Marvel Deck-Building Game*. Open the URL. Get a legal, fresh, collection-aware game setup in seconds. No install. No account. No tracking. Ever.

## What's in v1.0.4

This release is the biggest feature drop since launch: Second Edition solo mode is fully supported, a collapsible rules panel puts the right solo rules at your fingertips the moment a setup generates, multiplayer games now record a score per player, draw outcomes can be logged, and the Browse tab gains flexible sort order. Two bug fixes round it out — the Advanced Solo description now shows the correct card counts, and the home screen icon works correctly on iOS Safari.

---

### Second Edition solo mode (Standard v2)

Players using the Second Edition rulebook can now select **Standard Solo v2** as their play mode. It uses the same card counts as Standard and Advanced Solo (3 Heroes, 1 Villain Group, 1 Henchman Group, 25 Wounds), but applies the Second Edition-specific villain deck composition, scheme twist rules, mastermind ability transfer rule, and the one-tuck-per-turn cap. The mode is available for 1-player games only and does not inherit the scheme exclusions that apply to Standard Solo. All six locale files have been updated with full translations of the new mode label and description.

**Advanced Solo description corrected:** The mode selector was showing "4 Heroes and 2 Villain Groups" for Advanced Solo — the card counts used by a standard 2-player game, not solo. The correct counts (3 Heroes, 1 Villain Group) are now shown everywhere, including all six locale files. The generator itself was not affected; only the displayed description was wrong.

---

### Solo mode rules reference panel

After generating a setup in any solo mode with edition-specific rules (Standard Solo, Advanced Solo, or Standard Solo v2), a **Solo rules** collapsible section appears below the picked cards. It opens automatically so the rules are visible without any extra tap. The panel covers the key differences that matter at the table: villain deck composition, scheme twist effect, "each other player" scope, and any mode-specific rules such as the Master Strike cascade (Advanced Solo) or the mastermind ability transfer (Standard Solo v2). Collapse it once you're familiar; it stays collapsed for the session. The panel is hidden for Two-Handed Solo (no special solo rules apply) and for all multiplayer modes. All rules content is translated in all six supported languages.

---

### Per-player scores in multiplayer

Multiplayer game records (2–5 players) now store an individual score for each participant, along with an optional player name. When editing a game result in the History tab, a score row is shown per player — enter a name (or leave it blank to fall back to "Player 1", "Player 2", etc.) and their score. Game cards in the history list display each player's score inline (e.g. "Alice: 42 · Bob: 35"). Solo game records are unchanged and continue to store a single score value.

---

### Draw outcome

A third outcome — **Draw** — is now available when recording a game result. Use it when the mastermind's scheme completes its final effect but all players survive: a partial defeat that is neither a clean win nor a standard loss. Draw follows the same score rule as Loss — the score field is optional, and any value of 0 or greater is accepted. Game cards display "Draw" as a distinct label. The outcome is available for any player count. All six locale files include a translated draw label.

---

### Browse tab sort order

The Browse tab toolbar now includes a sort selector with three options: **Name** (A–Z, the existing default), **Release Year** (ascending, with A–Z as a tiebreaker for sets from the same year), and **In Collection** (owned sets first, then unowned, each group sorted A–Z). The active sort option is highlighted to match the type-filter button group. Sort order resets to Name on each page load and does not affect search or type-filter behaviour.

---

### Safari on iOS/iPadOS: home screen icon fixed

When adding the app to the home screen via **Share → Add to Home Screen** in Safari on iPhone or iPad, the icon was displaying as a screenshot of the page rather than the proper app icon. The root cause was an absolute path (`/icons/icon-192.png`) in the `apple-touch-icon` link tag, which resolved to the domain root and missed the `/random-legendary-llm/` sub-path the app is deployed under. The path has been corrected to the base-URL-aware form, consistent with the manifest link. Safari on iOS now shows the correct icon on the home screen.

> **Note on iOS/iPadOS install UX:** Safari does not display an automatic install banner or prompt. To add the app to your home screen, tap the **Share** button (box with upward arrow) in the Safari toolbar and select **Add to Home Screen**. Chrome and other third-party browsers on iOS are subject to Apple's WebKit restriction and do not support PWA installation on that platform regardless of app configuration.
